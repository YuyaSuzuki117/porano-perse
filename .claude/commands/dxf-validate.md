# /dxf-validate — DXF品質検証

生成したDXFファイルの品質とBlenderインポート適合性を検証する。

## 引数
- `$ARGUMENTS` — DXFファイルパス。省略時は最新の `output/drawings/*.dxf` を検証。

## 手順

### Step 1: ファイル読み込み
- ezdxf でDXFファイルを開く
- 読み込みエラーがあれば即座に報告

### Step 2: 構造チェック
Python スクリプトで以下を検証:
```python
import ezdxf
doc = ezdxf.readfile(dxf_path)
msp = doc.modelspace()
```

チェック項目:
- **レイヤー構成**: 必須レイヤー（壁/壁芯/建具/寸法/室名）の存在
- **エンティティ数**: 各レイヤーのエンティティ数を集計
- **壁の妥当性**:
  - ゼロ長さの壁がないか
  - 壁の長さが0.3m〜30mの範囲内か
  - 壁が閉じた多角形を形成しているか（部屋として成立するか）
- **寸法の妥当性**:
  - 寸法値と実際のジオメトリが一致するか
  - 単位がmmになっているか
- **開口部**: ドア幅600-1200mm、窓幅300-3000mmの範囲か

### Step 3: Blenderインポート互換性
- DXFバージョンがR2010以降か
- サポートされていないエンティティタイプがないか
- 座標系が適切か（原点付近にジオメトリがあるか）

### Step 4: レポート出力
```
=== DXF検証レポート ===
ファイル: output/drawings/example.dxf
バージョン: R2010

■ エラー (修正必須)
  - 壁レイヤーにゼロ長さの線分 (LINE #42)

■ 警告 (確認推奨)
  - 壁が閉じていません（開放端が2箇所）

■ 情報
  - レイヤー数: 9
  - エンティティ総数: 156
  - 壁: 24本、建具: 6個、寸法: 18個
  - バウンディングボックス: 8000mm × 6000mm

■ 次のステップ
  - エラーを修正して再実行、または `/perse-from-blueprint` へ進む
```
