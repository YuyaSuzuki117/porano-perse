リファクタリング実施: $ARGUMENTS

## 安全なリファクタリング手順

1. **事前分析**（Explore エージェント委任）
   - 対象ファイルの依存関係マップ作成
   - import/export の参照箇所を全列挙
   - 影響範囲の見積もり

2. **計画策定**
   - 変更ファイル一覧とその理由
   - リスク評価（High/Medium/Low）
   - 10ファイル超なら worktree エージェントに分割

3. **実装**
   - 型安全を維持しながら段階的に変更
   - 各ステップ後に `npx tsc --noEmit` で型チェック
   - import パスの更新漏れに注意

4. **検証**
   - TypeScript + ESLint + Build 全通過確認
   - Playwright MCP でビジュアル確認（3Dシーンが壊れていないこと）

## よくあるリファクタパターン
- **ストア分割**: useEditorStore (~2000行) → ドメイン別ストア
- **コンポーネント抽出**: 大きなコンポーネントの分割
- **共通ロジック抽出**: hooks/ への移動
- **型整理**: types/ への集約

対象が指定されていない場合はユーザーに確認。
