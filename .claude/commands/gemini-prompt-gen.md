Geminiに設計図面を読ませてClaude Codeへの制作指示プロンプトを生成する。ワンショットで図面→プロンプト→実行まで行う統合コマンド。

## 引数
- `$ARGUMENTS`: 図面画像パス [--style スタイル名] [--quality preview|draft|production] [--auto 自動実行]

## 手順

### 1. 図面画像確認
- 引数の画像パスをReadで表示
- 複数画像（平面図+立面図+参考写真）がある場合は全て読む

### 2. Gemini分析実行
既存の `/blueprint-analyze` と同等の分析を実行するが、出力をClaude Code実行形式に最適化する。

Gemini APIへのプロンプト:
```
あなたは建築設計の専門家です。添付の図面を分析し、以下の情報を正確に抽出してJSON形式で返してください。

必須項目:
1. 部屋の寸法（幅×奥行×天井高、mm単位）
2. 各壁の位置と仕上げ
3. ドア・窓の位置・サイズ・種類
4. カウンター・棚等の造作物
5. 床材・天井仕上げ
6. 寸法線から読み取れる全ての数値

推定が必要な場合は "estimated": true を付けてください。
```

### 3. プロンプト構築
Geminiの分析結果を、Claude CodeのBlenderパース制作に最適化した指示書に変換:

```markdown
# パース制作指示書

## 案件概要
- 案件名: {name}
- 用途: {usage}
- スタイル: {style}

## 部屋ジオメトリ（mm単位）
{壁・床・天井の詳細寸法}

## 開口部
{ドア・窓の位置とサイズ}

## 造作物
{カウンター・棚等}

## マテリアル指定
{床・壁・天井の仕上げ材}

## カメラプリセット
{推奨カメラアングル}

## 制作上の注意
- 天井・壁・床・建具を最優先で作る
- 家具はプレースホルダーでOK
- 余計な装飾は入れない
```

### 4. 自動実行（--auto指定時）
- 生成したプロンプトに基づき `/perse-from-blueprint` を自動実行
- preview品質でテストレンダリングまで一気通貫

### 5. 結果報告
- 指示書のパス: `output/blueprint-analysis/{name}_prompt.md`
- 分析JSON: `output/blueprint-analysis/{name}.json`
- --auto時: レンダリング結果画像を表示
