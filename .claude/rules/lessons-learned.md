# 案件知見の蓄積ルール

## いつ記録するか
- `/deliverable-pack` 実行時（案件完了）
- `/perse-quality-check` でスコア70点以上到達時
- `/pdf-dxf-pdca` で数値改善があった時

## 何を記録するか

### 案件完了時 → `output/projects/<案件>/lessons.md`
```markdown
## 案件: <案件名>
業種: <バー/カフェ/ホストクラブ/飲食店/オフィス/...>
図面形式: <PDF/JW_CAD/手書き/なし>
所要時間: <初回パース完成まで>

### うまくいったこと
- (具体的に)

### 失敗・手戻り
- (具体的に、原因と対策)

### 再利用できるもの
- マテリアル設定: (具体名)
- カスタムモデル: (具体名)
- スタイルJSON: (パス)
- カメラアングル: (座標)

### 次の同業種案件への申し送り
- (具体的に)
```

### PDF→DXF改善時 → ダッシュボード + メモリ
- どのパラメータ変更が効いたか（定量値）
- 図面の特徴（縮尺/線種/文字配置）と検出精度の関係
- 回帰テスト結果（過去の図面を壊していないか）

### 品質チェック時 → `output/projects/<案件>/quality_log.md`
- スコア推移（反復ごと）
- 効いた改善（カメラ/マテリアル/照明のどれか）
- 業種別の品質パターン

## 参照タイミング
- `/project-init` 実行時 → 同業種の過去lessons.mdを自動表示
- `/perse-from-blueprint` 実行時 → 同業種のマテリアル/カメラ設定を提案
- `/pdf-dxf-pdca` 実行時 → 過去の改善パターンを参照
