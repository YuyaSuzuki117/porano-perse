# PDF→DXF 読み取りプロトコル（最重要）

対象: `scripts/*-dxf.py` を手書きする全ケース

## 鉄則

1. **「概算」禁止** — 全寸法はPDFから正確に読み取る。読めない場合は `# TODO: 要確認` コメントを付けて仮値を使い、ユーザーに報告する
2. **欠落禁止** — PDFに描かれている全要素をDXFに含める。省略しない
3. **2パス読み取り** — 1回目: 全体構造抽出、2回目: 詳細検証

## 2パス読み取り手順

### Pass 1: 全体構造（まず木を見る）

1. **外形寸法の確定**
   - 上辺/下辺/左辺/右辺の全区間寸法を書き出す
   - 合計値を検算（区間合計 = 全体寸法）
   - L字/凸字/不整形の段差位置を特定

2. **室名と面積の全数カウント**
   - PDFに書かれている全ての室名をリストアップ
   - 面積表記があれば記録
   - 「全室面積合計 ≈ 建物面積」の検算を行う

3. **建具の全数カウント**
   - ドア: 開き円弧/引戸マークを全数カウント
   - 窓: ダブルライン/W表記を全数カウント
   - 開口: 破線表記を全数カウント
   - **カウント結果をスクリプト冒頭にコメントで記録**

4. **什器・設備の全数カウント**
   - テキストラベルのある什器を全数カウント
   - 設備記号（消火栓/分電盤/給排水等）を全数カウント

### Pass 2: 詳細抽出（次に枝を見る）

5. **壁の座標抽出**
   - 外壁: Pass 1の寸法から座標を計算
   - 内壁: 各間仕切りの始点/終点を寸法線から特定
   - **全壁をリスト化し、Pass 1の室名カウントと照合**
   - 「N室の壁 = N個の閉じた多角形」になるか検証

6. **建具の詳細**
   - 各ドア/窓がどの壁に属するか特定
   - 位置（壁の始点からの距離）、幅、高さを記録
   - W600/W900/W1000 等の表記を見逃さない
   - **Pass 1のカウント数と照合**

7. **什器の配置**
   - 中心座標、幅×奥行、回転角を記録
   - **バーなら「カウンター」、飲食店なら「厨房設備」が必須** — 主要什器の欠落は致命的

8. **展開図からの高さ情報**
   - 各室の天井高を展開図から正確に読み取る
   - 造作物（棚/カウンター等）の高さ寸法を読み取る

## スクリプト構造テンプレート

```python
"""
{案件名} PDF → DXF 変換
出典: {PDFファイル名}
設計: {設計事務所名}
縮尺: {縮尺}

=== Pass 1 読み取り結果 ===
外形: {幅}mm x {高さ}mm ({形状: 矩形/L字/不整形})
室数: {N}室 ({室名リスト})
建具数: ドア{N}個、窓{N}個、開口{N}個
什器数: {N}個
面積合計: {合計}㎡ / 建物面積{面積}㎡ = {比率}%

=== Pass 2 検証 ===
壁数: {N}本 → {N}室を囲む閉多角形になるか: {OK/NG}
建具: Pass1カウント{N}個 = スクリプト内{N}個: {OK/NG}
什器: Pass1カウント{N}個 = スクリプト内{N}個: {OK/NG}
主要什器チェック: {カウンター/厨房/レジ}: {OK/NG}
"""
```

## 検証チェックリスト（スクリプト完成後）

生成したDXFについて以下を確認:

- [ ] 全室が壁で囲まれているか（開いた壁がないか）
- [ ] Pass 1でカウントした建具数 = DXF内の建具数
- [ ] Pass 1でカウントした什器数 = DXF内の什器数
- [ ] 天井高にTODOが残っていないか
- [ ] 寸法線の区間合計が外形寸法と一致するか
- [ ] 「概算」コメントが残っていないか（残っている場合はユーザーに報告）

## 禁止事項

- 「この部屋は小さいから省略」→ 禁止。全室を含める
- 「このドアは重要じゃないから省略」→ 禁止。全建具を含める
- 「概算で十分」→ 禁止。PDFから正確に読み取る
- 「展開図は後で見る」→ 禁止。平面図と展開図を同時に読む
