# パース生成の優先順位ルール

対象: `scripts/blender/**/*.py`, `scripts/render-template.py`, `scripts/template-to-json.ts`

## 一発目のパース生成（初回レンダリング）

初回パース生成時は以下の優先順位に従う。
建築の箱（エンベロープ）を先に固め、家具は後から差し替える前提で進める。

### 最優先: 天井・壁・床・建具をしっかり作る
- **壁**: 正しい寸法、巾木（ベースボード）、適切なマテリアル
- **天井**: 廻り縁（クラウンモールディング）、正しい高さ
- **床**: スタイルに合った床材（テラゾー/木目/タイル等）
- **建具**: ドア・窓のフレーム、ガラス、位置精度
- **造作物**: カウンター、棚等の固定什器は構造をしっかり
- 図面がある場合は寸法を正確に合わせる

### 後回し: 家具・設備はこだわらない
- 家具はプレースホルダー品質でOK（位置と大きさが合っていれば十分）
- 後から単体でしっかりモデリングしたものを差し替える前提
- GLBフォールバック（プリミティブ）でも初回は問題なし
- カスタムモデルがあれば使うが、なくても止めない

### 禁止: 余計な線や物を絶対に入れない（最重要）
- **Blender新規テンプレートはシーンを完全にクリアしてから構築する**（デフォルトCube/Camera/Light削除必須）
- **`init_scene()` は全オブジェクト削除から開始**。Blenderデフォルトのキューブ・カメラ・ライトを残さない
- 装飾的なディテール（額縁・観葉植物・小物等）は初回では絶対に入れない
- 複雑なエフェクト（パーティクル、ボリュメトリクス、グレア等）は初回不要
- 参照画像・図面にない要素を勝手に追加しない
- サンプル家具やダミーオブジェクトを「雰囲気づくり」で置かない
- 後から練っていくので、最初は建築の箱だけに徹する
- **図面に描かれていないものは存在しないものとして扱う**

## 修正・改善の進め方

Codex iteration-checklist.md に準拠:
1. カメラ位置を合わせる
2. 全体比率を確認
3. 主要曲線と接続部
4. 脚のテーパとスタンス
5. 材質の色とroughness
6. 細部ディテール

## 家具の差し替えフロー

1. 初回パースで配置・スケール感を確認
2. 個別に `scripts/blender/models/` で高品質モデルを作成
3. `furniture_importer.py` の `_custom_generators` にルーティング追加
4. 再レンダリングで確認
