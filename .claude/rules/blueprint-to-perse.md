# 図面→パース変換ルール

対象: `scripts/render-*.py`, `scripts/blender/blueprint_converter.py`, `output/blueprint-analysis/**`

## ワークフロー

```
設計図面 → Gemini Vision分析 → 構造化JSON → Blenderスクリプト生成 → レンダリング
```

## Gemini図面分析のルール

### 必ず抽出する項目
1. **部屋寸法**: 幅×奥行×天井高（mm単位）。寸法線があれば厳守。
2. **壁の位置**: 各壁のstart/end座標。非矩形の部屋も正確に。
3. **開口部**: ドア・窓の位置/サイズ/タイプ。框・枠の形状も可能なら。
4. **造作物**: カウンター・棚等は建築に固定されたもの全て。
5. **床・天井仕上げ**: 素材名だけでなく色味・パターンも。

### 推定が必要な場合
- 寸法線がない場合はドアサイズ（一般的に幅900mm×高2100mm）を基準に推定
- 推定値には必ず `"estimated": true` フラグを付ける
- 推定の根拠を `notes` フィールドに記録

### 複数図面がある場合
- 平面図 > 立面図 > 展開図 > パースの優先順位で情報を統合
- 矛盾がある場合は平面図を優先

## Blenderスクリプト生成のルール

### 座標系の統一
- 図面: X=右, Y=上（2D平面図）
- Blender: X=右, Y=奥, Z=上
- 変換: 図面(x,y) → Blender(x, y, 0) で床面配置、高さはZ

### 寸法の扱い
- JSONではmm単位で格納
- Blenderスクリプト内でm単位に変換（÷1000）
- 壁厚のデフォルト: 120mm（図面に明記なければ）

### スクリプト構成
1. `init_scene()` でクリーンスタート
2. 壁→床→天井の順で部屋の箱を作る
3. 開口部をブーリアンカットまたは分割で表現
4. 造作物を配置
5. 家具はプレースホルダー（後から差し替え）
6. 照明→カメラ→レンダリング設定

### 禁止事項
- `bpy.ops.mesh.*` の使用（bmesh必須）
- GUI依存の操作（`--background` 実行前提）
- 未確認の寸法でのレンダリング（推定値は必ず明記）
- 参照画像にない要素の追加
