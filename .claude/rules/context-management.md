# コンテキスト管理ルール（最重要）

このプロジェクトは 200+ ファイル・大規模3Dコードベース。
コンテキスト溢れ = セッション死。以下を厳守。

## 読み込み制限
- 1セッションでメインコンテキストに読むファイルは **10ファイル以内**
- 500行超のファイルは `limit` パラメータで必要部分のみ
- `useEditorStore.ts` (~2000行) は絶対に全読みしない → 該当セクションのみ
- `node_modules/`, `.next/`, `.claude/worktrees/` は読まない

## エージェント委任（必須）
- **調査**: Explore エージェント → 結果のみ受け取る
- **マルチファイル変更**: worktree 隔離エージェントに委任
- **10ファイル以上の変更**: 必ずエージェント分割（5ファイル/エージェント目安）
- **ビルド/テスト**: バックグラウンド実行

## 出力のスリム化
- 長い分析結果 → メモリファイルに書き出し、コンテキストに残さない
- ツール結果が 100行超 → 要約してから次のステップへ
- git diff/log の出力は最小限（`--oneline -n 8` 等）

## セッション継続性
- 機能完了ごとに完了報告 → ユーザーの `/clear` 判断を待つ
- 中断リスクがある大タスクは開始前にメモリファイルに計画を保存
- 再開時は `git log --oneline -n 8` + メモリ参照で状態復元
