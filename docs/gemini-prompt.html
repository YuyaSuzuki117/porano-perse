Gemini 図面読み取りプロンプト

PDFの平面図をGeminiに見せて、gen-dxf.py互換のJSONを生成させるためのプロンプト

使い方
1
gemini.google.com を開く
2
PDFファイル（または平面図の画像）をアップロード
3
下のプロンプトをコピーして送信
4
返ってきたJSONを output/blueprint-analysis/<案件名>.json に保存
5
python scripts/gen-dxf.py --json <JSONパス> -o <出力DXF> で図面生成

※ 複数ページのPDFは平面図のページだけスクリーンショットして送るのが確実です

MAIN 図面読み取りプロンプト（コピペ用）
あなたは建築図面の読み取りエキスパートです。 添付した平面図から以下の情報を正確に読み取り、JSON形式で出力してください。 ## 読み取りルール（厳守） 1. **単位はすべてmm（ミリメートル）** 2. **座標系**: 原点=図面の左下角、X=右方向、Y=上方向 3. **縮尺**: 図面の縮尺表記（1:50等）を確認し、寸法線の値を実寸mmで記録 4. **推測禁止**: 寸法線が読めない場合はドアサイズ(幅900mm)を基準に推定し、"estimated": true を付ける 5. **省略禁止**: 図面に描かれている全ての壁・建具・什器・部屋を含める ## 読み取り手順 ### Pass 1: 全体把握 - 外形寸法（幅×奥行）を寸法線から読み取る - 全室名をリストアップ - 全建具（ドア・窓・引戸・折戸・開口）を数える - 全什器・設備を数える ### Pass 2: 詳細抽出 - 各壁の始点・終点座標を寸法線から正確に算出 - 各壁上の建具の位置・幅・タイプを記録 - 各什器の位置・サイズを記録 ## 出力JSON形式 ```json { "source": "gemini-vision", "pdf_file": "（ファイル名）", "scale_detected": "1:50", "project_name": "（案件名があれば）", "walls": [ { "id": "W1", "start_x_mm": 0, "start_y_mm": 0, "end_x_mm": 12000, "end_y_mm": 0, "thickness_mm": 120, "type": "exterior", "openings": [ { "type": "door", "position_mm": 3000, "width_mm": 900, "height_mm": 2100 }, { "type": "window", "position_mm": 6000, "width_mm": 1800, "height_mm": 1200, "sill_mm": 800 }, { "type": "sliding_door", "position_mm": 9000, "width_mm": 1600, "height_mm": 2100 } ] } ], "rooms": [ { "name": "エリアA", "wall_ids": ["W1", "W2", "W3", "W4"], "area_m2": 35.0, "center_mm": [6000, 4000], "polygon_mm": [[0,0], [12000,0], [12000,7000], [0,7000]] } ], "fixtures": [ { "name": "カウンター", "x_mm": 3000, "y_mm": 5000, "width_mm": 2400, "depth_mm": 600, "rotation_deg": 0, "estimated": false } ] } ``` ## 各フィールドの説明 ### walls（壁） - **id**: "W1", "W2", ... の連番 - **start/end_x_mm, start/end_y_mm**: 壁の中心線の始点・終点（実寸mm） - **thickness_mm**: 壁厚（外壁: 150-200mm, 内壁: 100-150mm, 間仕切り: 65-100mm） - **type**: "exterior"（外壁）/ "interior"（内壁）/ "partition"（間仕切り） - **openings**: この壁上の建具リスト - **type**: "door"（開き戸）/ "sliding_door"（引戸）/ "folding_door"（折戸）/ "window"（窓）/ "opening"（開口） - **position_mm**: 壁の始点からの距離 - **width_mm**: 建具の幅 - **height_mm**: 建具の高さ（ドア: 2100mm, 窓: 1200mm） - **sill_mm**: 窓台の高さ（窓のみ、通常800mm） ### rooms（部屋） - **name**: 室名（図面のテキストそのまま） - **wall_ids**: この部屋を囲む壁のIDリスト（時計回り） - **area_m2**: 面積（図面に記載があればその値、なければ計算値） - **center_mm**: 部屋の重心座標 - **polygon_mm**: 部屋の頂点座標リスト（時計回り、閉じなくてOK） ### fixtures（什器・設備） - **name**: 什器名（図面のテキストそのまま） - **x_mm, y_mm**: 左下角の座標 - **width_mm, depth_mm**: 幅と奥行 - **rotation_deg**: 回転角度（0/90/180/270） - **estimated**: サイズが推定の場合 true ## 重要な注意 - **壁は必ず閉じたループを形成すること** — 各部屋を囲む壁が途切れないように - **寸法線の区間合計が外形寸法と一致するか検算**すること - **展開方向図や凡例エリアの要素は含めない**（平面図部分のみ） - 室名に面積記載がある場合（例: "エリア(35帖)"）、そのまま name に入れる - 日本語テキストはそのまま出力（変換不要） - L字型・凸字型の部屋は polygon_mm で正確に形状を表現 JSONのみ出力してください。説明文は不要です。
CHECK 検証プロンプト（JSONの修正依頼用）
先ほど出力したJSONを以下の観点で検証・修正してください。 ## 検証チェックリスト 1. **壁の閉合チェック**: 各部屋のwall_idsの壁が閉じたループを形成しているか？ - 壁Aの終点と壁Bの始点が一致（または200mm以内）しているか確認 - 閉じていない場合、不足する壁を追加 2. **寸法検算**: - 外形の各辺: 区間寸法の合計 = 全体寸法になっているか？ - 計算が合わない場合は修正 3. **建具の全数チェック**: - 図面上のドア（開き円弧の記号）を全数カウント → walls.openingsの合計と一致するか？ - 引戸（二重線記号）、窓（ダブルライン/W表記）も同様にチェック 4. **室名の全数チェック**: - 図面上の全室名テキスト → rooms配列と1:1対応しているか？ - 不足する部屋があれば追加 5. **polygon_mmの妥当性**: - 各部屋のpolygon_mmの面積を計算し、area_m2と整合するか？ - 帖数表記がある場合: 1帖=1.6562m² で検算 修正後のJSONを出力してください。
CMD JSON保存後のコマンド
